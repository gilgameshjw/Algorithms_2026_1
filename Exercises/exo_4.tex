\begin{problem}[Courses Allocation: Dynamic Programming VS Greedy Computing!] 
For conflict free course allocation as in the course: \\
Consider set of activities:
\[
    S = \{a_1 ,\, a_2 ,\, \cdots ,\, a_n\}, 
\]
each with starting and ending times $s_i$, $f_i$.
Assume also that $S$ is sorted such that:
\[f_1\leq f_2\leq \cdots \leq f_n\]

A set of compatible activities $\hat S_{ap}$ means that each activities of $\hat S_{ap}$ starts after the last activity ended and ends before the next activity starts.


\begin{itemize}
    \item {\bf Dynamical Approach:} \\ Consider the recurrence below for the cost $c[a, p] = c[a, k] + c[k, p] + 1$ between the course $a_a$ and $a_p$:
    \[ c[a, p] = 
\begin{cases}
0 & \text{if } \hat S_{ap} = \emptyset, \\
\max \left\{ c[a, k] + c[k, p] + 1 \mid a_k \in \hat S_{ap} \right\} & \text{if } \hat S_{ap} \neq \emptyset,
\end{cases}\]

\begin{enumerate}
    \item  Structuring the problem with $\hat S_{ap}$ actually characterises optimal substructures? Can therefore dynamical computing can be used? 
    \item  Explain/draw how that can be used for dynamical programming, top down with recurrence: $RecuSelect(s,\, f,\, k,\, n)$, starts $(s,\, f,\, 0,\, n)$, return list $a_i$'s.
    \item dynamical programming with tabulation. 
\end{enumerate}

\item {\bf Greedy Approach:} 
\begin{enumerate}[I.]
    \item What is the greedy choice for the activity-selection problem? \footnote{choose the activity in S
with the earliest finish time and bottom up, like in the course...:-)}
    \item Write the pseudocode for the greedy approach $\mathrm{GreedySchedule}(s, f, n)$ \\
	    {\bf !!!consider sorting courses first!}.
    \item Prove that your GreedySchedule is optimal using\footnote{See course V! } ({\bf hard})
    \begin{itemize}
        \item induction
        \item stay ahead arguments
        \item contradiction
    \end{itemize}
\end{enumerate}
\end{itemize}


\end{problem}
